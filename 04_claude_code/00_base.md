## 🚀 第一部分：核心命令

### 1.1 `/init` - 初始化对话环境

**📖 功能说明**

`/init` 命令会创建一个对话记录文件，记录你和Claude的所有对话内容。每次发送消息时，Claude都会根据选择的模式处理对话记录，你可以随时查看文件回顾对话历史。

---

#### 📂 三种常见的文档类型

##### 🟠 CLAUDE.md - 项目级配置文件

| 属性            | 内容                                                             |
| --------------- | ---------------------------------------------------------------- |
| **📍 存储位置** | 项目目录                                                         |
| **🎯 主要用途** | 通过 `/init` 生成，记录项目特定的对话历史                        |
| **✨ 核心特性** | • 提交到源代码控制<br/>• 与其他工程师共享<br/>• 项目特定配置记录 |

##### 🟡 CLAUDE.local.md - 本地个人配置文件

| 属性            | 内容                                                                 |
| --------------- | -------------------------------------------------------------------- |
| **📍 存储位置** | 项目目录                                                             |
| **🎯 主要用途** | 存放个人指令和Claude的自定义配置                                     |
| **✨ 核心特性** | • 不与他人共享<br/>• 个人对Claude的特殊说明<br/>• 本地测试和临时记录 |

##### 🟢 ~/.claude/CLAUDE.md - 全局配置文件

| 属性            | 内容                                                       |
| --------------- | ---------------------------------------------------------- |
| **📍 存储位置** | 用户主目录下的 `.claude` 文件夹（`~/.claude/`）            |
| **🎯 主要用途** | 应用于您计算机上的所有项目                                 |
| **✨ 核心特性** | • 全局通用指令配置<br/>• 跨项目统一设置<br/>• 个人偏好保存 |

---

#### 🔄 三种不同的处理模式

| 模式                    | 行为说明                                               |
| ----------------------- | ------------------------------------------------------ |
| **📝 追加模式（默认）** | 每次发送消息，将内容**追加**到文件末尾，保留完整历史   |
| **♻️ 覆盖模式**         | 每次发送消息，**覆盖**整个文件，只保留最新的对话       |
| **🗑️ 清空模式**         | 每次发送消息，**清空**文件内容，仅保留最新的消息和回复 |

**⚙️ 命令示例**

```bash
# 使用各种模式创建文件
/init                      # 📝 默认追加模式
/init overwrite            # ♻️  覆盖模式
/init clear                # 🗑️  清空模式
/init local                # 🟡 本地配置模式
```

#### ✏️ 在claude.md中快速添加内容

你可以通过两种方式快速编辑和添加内容：

**方式 1️⃣ - 直接编辑文件**

打开 `CLAUDE.md` 使用 Markdown 语法自由组织内容：

```markdown
# 项目配置

## 数据库设置

- PostgreSQL 13
- 连接池大小：20

## API 集成

- Gemini API ✓
- OpenAI API 进行中
```

**方式 2️⃣ - 终端快速标记**

在对话中使用 `#` 符号快速添加内容，自动保存到 `claude.md`：
这种方法可以让claude来帮你添加内容，你可以只输入你的简单意图。

**方式 3️⃣ - 使用 `@` 指定文件**

当你需要操作非默认文件（例如项目中的多个会话或笔记）时，可用 `@` 前缀指向目标文件。

```bash
@design.md 添加一个新的架构图说明
/compact @notes.md               # 紧凑化 notes.md
/save @meeting_minutes           # 将当前对话保存到指定文件
```

这个约定适用于所有支持文件名参数的命令或输入，使你可以在同一项目内管理多个对话文档。

> ⚙️ **代码规划与编辑模式**
>
> 当处于“planning”或编程交互模式时，`@` 前缀同样有效，指向任意源码文件。Claude 会将内容直接写入指定文件，而不仅限于 `claude.md`。
>
> 例如：
>
> ```bash
> @src/app.py  # 表示后续内容将修改或创建这个 Python 文件
> def new_feature():
>     """TODO: 添加业务逻辑"""
>
> /plan @src/app.py 添加错误处理逻辑
> /edit @src/app.py 删除未使用的 import
> ```
>
> 这使得开发过程更自然：你可以在对话中描述需求，同时引用具体文件，Claude 会自动在工作区中更新它们。

```bash
# 重要决策
这次对话中采用了 TypeScript 实现核心模块

# Bug 修复方案
数据库连接超时通过增加连接池大小解决
```

### 1.4 `/compact` - 紧凑对话文件（专家指南）

**🧹 功能说明**

`/compact` 命令对现有的对话记录文件（默认是 `claude.md`）执行格式整理。它不会改变对话内容的实际文字，只会：

- 修剪多余的空行
- 删除重复的分隔标记（例如连续的 `---`）
- 清理自动生成的备份注释或时间戳碎片
- 可选地将结果输出为 `filename.compact.md` 以保留原文件

这一命令属于“维护/排版”类别，不会影响Claude的上下文记忆或对话状态。

> ⚠️ `/compact` 不会删除任何消息，也不会清除对话历史；它只是让文件**看起来更紧凑**。

**📋 使用方式**

```bash
/compact                   # 压缩当前工作目录下的 claude.md
/compact other.md          # 对指定文件执行同样操作
/compact other.md --inplace # 覆盖原文件（如果支持）
```

在绝大多数情况下，只需不带参数运行即可。

**✨ 典型效果**

- ✨ 去掉多次回车或空白行
- 🧩 合并相邻相同标题（例如两次复制粘贴的时间标记）
- 📦 减少文件体积，有助于 Git 提交或邮件分享
- 🗂️ 生成带 `.compact.md` 后缀的备份版本，便于审查

#### 🧠 为什么 `/compact` 很少用？

- 它不是必需命令，只是文件排版工具；对对话逻辑或Claude记忆没有任何影响。
- 大多数用户习惯直接编辑 Markdown 或让 `/save` 产生格式良好的输出。
- `/reset` 和 `/init clear` 真正会改变对话状态，用户更常用它们来管理会话，而不是整理文件。

因此，请在准备**提交/发布/归档**时运行它，而不是在日常交互中频繁执行。

#### 🧩 与 `/reset` 和 `clear` 的区别

| 命令          | 作用范围         | 对话上下文  | 文件内容                 | 常见用途                 |
| ------------- | ---------------- | ----------- | ------------------------ | ------------------------ |
| `/reset`      | Claude 环境      | ✔️ 清除     | 不影响                   | 开启全新对话线程         |
| `/init clear` | 新会话初始化模式 | ✔️ 清空文件 | 清空旧内容，仅保留新消息 | 启动时丢弃历史           |
| `/compact`    | 仅文件格式化     | ❌ 无影响   | 仅去冗余、压缩格式       | 提交前整理、减少仓库噪音 |

> 💡 注意：`/compact` 从不改变Claude“记住”的内容，而 `reset`/`clear` 则彻底重置记忆状态。它们解决的是完全不同的问题！

#### ✅ 最佳实践

1. **提交/共享之前**：在将 `claude.md` 提交到 Git 或发送给同事之前运行 `/compact`，即可获得干净的历史记录。
2. **归档对话**：打算将对话保存为长期档案时，先 compact 再 `mv` 到 `archive/`。
3. **自动化脚本**：如果你在 CI 中包含对话文件，可在流水线里自动调用 `/compact` 以降低 diff 噪音。
4. **Git 自动提交**：`/compact` 只是格式整理，若想将结果纳入版本控制，只需在同一条命令链中加上 `git add` 和 `git commit`。与 `compact` 无关，这只是一个普通的 Git 提交流程，写成别名或脚本即可提高效率。
5. **避免滥用**：不需要在每次对话后执行；它不会释放 Claude 上下文或减轻模型负担。

示例：

```bash
# 压缩并提交
/compact && git add claude.md && git commit -m "tidy conversation"
```

#### 🔁 自动提交提示

大多数 Claude 客户端在执行 `/compact` 时会**自动调用 Git 提交**，无需你手动运行 `git add`/`commit`。
提交信息通常为类似 `"[claude] compacted claude.md"` 或根据当前文件名自带结构；你也可以在配置中自定义模板。

> 📌 这个行为只是客户端在本地跑的脚本，与 `/compact` 命令本身无关——命令只负责整理文件，提交是自动化的额外步骤。

### 1.6 规划/编辑模式 & 快捷入口

**✏️ 功能说明**

规划模式（也称为编辑模式）是与 Claude 进行代码或文档修改时使用的上下文。它允许你把对话中的指令直接映射到工作区内的文件。你可以手动输入相关命令，也可以使用快捷键启动。

**🔑 快捷键**

- 在多数客户端中，连续按两次 **`Shift+Tab`** 可以直接进入规划模式（无需打开命令面板）。
  - 第一次 `Shift+Tab`：切换到自动接受模式（`⏵⏵ accept edits on`）
  - 第二次 `Shift+Tab`：切换到规划模式（`⏸ plan mode on`）
- 部分 IDE 插件也允许通过 `Ctrl+Shift+P` / `Cmd+Shift+P` 搜索 `/plan`，但 `Shift+Tab` 是最常见的快速键。
- 在 CLI/终端中直接输入 `/plan` 或 `/edit` 即可进入规划模式。
- 进入后，提示符会变为 `> plan:` 或类似标识，表示后续内容将被视为对文件的更改请求。

**📋 使用示例**

```bash
/plan                       # 切换到规划模式
@src/utils.js              # 指定目标文件
添加一个新的工具函数模板
/exit                       # 退出规划模式
```

已经介绍过的 `@` 文件前缀在此处同样适用，能够定位任意源码文件进行修改。

> 💡 如果你通过 IDE 插件工作，通常可以右键文件并选择“与 Claude 规划”之类的选项，此操作会自动打开规划模式并将 `@` 前缀填入。

**✨ 好处**

- 让讨论和代码修改无缝衔接
- 避免在编辑器和对话窗口之间来回切换
- 支持单次对话中对多个文件并行操作

## ❓ 第三部分：常见问题 (FAQ)

### Q1: 如何快速切换不同的处理模式？

**A:** 使用相应的参数调用 `/init` 命令即可快速切换。

```bash
# 从追加模式切换到覆盖模式
/init overwrite

# 切换到清空模式
/init clear

# 使用本地配置模式
/init local
```

---

### Q2: 保存的文件在哪里？

**A:** 使用 `/save` 命令时的文件位置：

- **默认位置**：当前工作目录
- **指定名称**：`/save filename` 后会以 `filename.md` 保存
- **查找文件**：可以通过文件管理器或终端查找

```bash
# 查找最近保存的文件
ls -lt *.md | head -5
```

---

### Q3: 如何恢复被清空的对话？

**A:** 若使用了 `/reset` 或 `/init clear` 命令，**无法恢复**。

**预防措施：**

```bash
# 在重要操作前使用 /save 备份
/save backup_before_reset

# 然后进行其他操作
/reset
```

---

### Q4: 我的上下文太大了，想提取重要信息做压缩怎么办？

**A:** Claude自身没有自动“压缩”模型记忆的命令，`/compact`也仅清理文件格式。要从庞大的对话中抽取精华，可采用如下策略：

1. **手动摘录或重写摘要**：
   - 使用 `/save` 导出当前对话为文件，然后手工删除噪音，仅保留关键点。可以在文件中使用 `#` 标记来辅助筛选。
   - 或者在对话中输入类似 `请总结上一段内容的关键要点` 让Claude生成摘要，并复制到新文件中。

2. **分割会话**：
   - 通过 `/reset` 开始新会话，保留旧会话的总结作为新会话开头，这样上下文长度便能控制在合理范围。
   - 例如：
     ```bash
     # 先让Claude写一个简短摘要
     # “总结：我们讨论了X、Y和Z，下步计划是…”
     /reset
     # 在新会话中粘贴该摘要作为起点
     ```

3. **使用外部工具**：
   - 将导出的 Markdown 交给类似 `pandoc`、`grep`、`sed` 或在线摘要服务进一步处理。
   - 结合 `git diff` 和 `compact` 在版本控制中保留关键更改。

> 💡 关键思想是“人为剪辑＋分段重启”，而不是期待某个命令自动压缩模型上下文。
